What is Continouous Integeration?

In software engineering, continuous
integration (CI) is the practice of merging
all developer working copies to a shared
mainline several times a day
WIKI

The basic idea is that repo should always have a version of your project that is functional.
Continouous Integeration shows its value for test-driven development.

Types of tests

  • Unit Testing
  • Testing small pieces of code like individual functions.

    • Example: Testing your square root function for your calculator project.


  • Integration Testing
  • Testing to see if different parts of your project work together.

    • Example: Testing to see if different modules work with each other such as your flask app, database, and javascript work alongside each other. Click here for more.


  • Functional Testing
  • Testing the functionality of the application either manually or with automation tools.

    • Example: Testing if your project does everything as intended. Click here for more.


Travis CI

We recommend using Travis CI because...

  • Works with github and is part of github education pack
  • Free to use for public repositories on github
  • Supports open source projects

Travis CI automatically create virtual environments for testing and running all the tests specified by the users and send email notifications

Setting up Travis CI

  1. Logging into travis-ci.org with your github credentials.
  2. Toggle the switch next to your project repo.
  3. Configure Travis by adding a .travis.yml file to your project repo.

    1. In this file, we will specify to Travis that the language of our project and which versions of the language.
      For our example, we will be testing our project in Python version 2.7 and Python version 3.6
      2. language: python

      python:
      - 3.6
      - 2.7

    2. Next, we tell Travis that we are testing our project in Ubuntu
      3. os:
      - linux

    3. Once we have that we need to tell Travis how to setup our project.
      4. install:
      - pip install -r requirements.txt

    4. Finally we need to tell Travis what tests to run on the virtual machine created.
      For example, we will be using pytest to perform unit tests.
      5. script:
      - pytest

    At the end our .travis.yml file should look this.

    language: python

    os:
    - linux

    python:
    - 3.6
    - 2.7

    install:
    - pip install -r requirements.txt

    script:
    - pytest
